Employee Analytics Database

A PostgreSQL-based Employee Analytics Database designed to store, manage, and analyse employee information across multiple departments. This project demonstrates database design, SQL querying, and business intelligence concepts using real-world HR analytics scenarios.



Project Overview

This project simulates an employee management system for an organization. It stores employee, department, performance, attendance, and attrition data and uses SQL queries to generate valuable business insights.

The database follows normalization principles and uses relational tables connected through primary and foreign keys.



Objectives

- Design a normalized relational database
- Store employee and department information
- Track employee performance and attendance
- Monitor employee attrition
- Generate business insights using SQL
- Practice PostgreSQL database development



Database Schema

The database consists of the following tables:

- Departments
- Positions
- Employees
- Performance
- Attendance
- Attrition


Relationships

- One Department has many Employees
- One Position can belong to many Employees
- One Employee has one Performance record
- One Employee has one Attendance record
- One Employee has one Attrition record



Business Questions Answered

The project includes SQL queries that answer common HR analytics questions such as:

- Average salary per department
- Top 10 highest-paid employees
- Employee count per department
- Average performance score by department
- Employee attrition rate
- Departments with the highest employee attrition
